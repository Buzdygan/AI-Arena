Contest

The contest is scheduled as round-robin tournament, which means that a bot plays with all other bots exactly once. He gains 2 points for win, 1 for draw and 0 for loss. The position in a ladder depends on total sum of points. Each user can submit at most one bot source files (WARNING: sending next source file will override any previoulsy submitted files!).

Pok game

The game called Pok is one-on-one simple version of Texas Hold'em Poker. The rules are following:
There are two players and every deal each of them receives one private card. Then they bet twice and the winner gets points, which are in the final stake. The goal is to gain more points then the opponent, what can be achieved by betting cautiously, depending on the card and opponent behaviour. A draw is possible if players reached the same amount of points.

The game consists of 200 deals. Here is precise description of a deal:

  1. Each player receives one random card 0, 1, 2, 3 or 4.
  2. First bet:
    • Players bet simultaneously and secretly, no less then 10 and no more then 200.
    • If one of the players bets less then his opponent, he can fold(surrender the stake) or call(agree on higher stake).
    • If he folds, this is the end of the deal and his opponent gains the lower stake.
    • If he calls, the stake is raised and deal continues.
  3. Second bet:
    • Only if players agreed on the stake in first bet.
    • Runs in the same way as first, and the stake is added to first one.
  4. Showdown:
    • Only if players agreed on the stake in second bet.
    • Players show their cards. The one with higher card gains the stake. In case of draw nobody gains any points.

Protocol

A bot program should write on standard output and read from standard input. Every message he sends consists of one integer number followed by endline character. Messages sent to bot may contain one or more integers separeted by whitespace characters, ended by endline character. During a single deal bot exchanges following numbers in this order:

  1. Receives a number(from 0 to 4) indicating his private card.
  2. First bet:
    • Sends his bet(from 10 to 200).
    • Receives a number indicating a bet of his opponent.
  3. Then, depending on if his opponent's bet is:
    • higher - if bot is going to call skips to the next step, else(folding) sends 0 finishing a deal.
    • lower - receives 0 or 1. 0 means that his opponent folds and this is the end of a deal, 1 means that opponent calls and a deal continues.
    • equal - then just goes to next step.
  4. Second bet. Looks exactly like steps 2 and 3. If he or his opponent folds, a deal is finished.
  5. Showdown. If players agreed on a stake value, receives a number indicating his opponent's card.
A single game consists of 200 deals in a row. If bot exceeds time or memory limit, or does not follow the protocol, he loses the game.

Limits

Limits for bot, which oblige during each game, are: USTALIC for memory and USTALIC for time.
